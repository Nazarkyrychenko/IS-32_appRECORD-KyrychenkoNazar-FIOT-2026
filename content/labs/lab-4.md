## 2. Виконання завдань з програмним кодом

### Завдання 1. Встановлення бібліотек
У терміналі виконано команду для додавання модулів логування та обробки файлів:
```bash
npm install morgan winston multer
```

### Завдання 2. Налаштування професійного логування (Winston)
Створено систему логування у файли для розділення звичайних подій та помилок.

```javascript
// utils/logger.js
const winston = require('winston');

const logger = winston.createLogger({
    level: 'info',
    format: winston.format.combine(
        winston.format.timestamp(),
        winston.format.json()
    ),
    transports: [
        new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
        new winston.transports.File({ filename: 'logs/combined.log' })
    ]
});
```

### Завдання 3. Логування HTTP-запитів (Morgan)
Інтегровано Morgan для автоматичного виводу HTTP-запитів у консоль розробника.

```javascript
// app.js
const morgan = require('morgan');
app.use(morgan('dev'));
```

### Завдання 4. Завантаження та валідація файлів (Multer)
Налаштовано збереження файлів з кастомними іменами та перевіркою типу/розміру.

```javascript
// middleware/upload.js
const multer = require('multer');

const storage = multer.diskStorage({
    destination: (req, file, cb) => cb(null, 'uploads/'),
    filename: (req, file, cb) => cb(null, Date.now() + '-' + file.originalname)
});

const fileFilter = (req, file, cb) => {
    if (['image/jpeg', 'image/png', 'application/pdf'].includes(file.mimetype)) {
        cb(null, true);
    } else {
        cb(new Error('Недопустимий формат файлу'), false);
    }
};

const upload = multer({ 
    storage, 
    limits: { fileSize: 2 * 1024 * 1024 }, // 2MB
    fileFilter 
});
```

### Завдання 5. Ендпоінти для завантаження файлів
Реалізовано маршрути для завантаження одного та кількох файлів одночасно.

```javascript
// app.js
app.post('/api/upload', upload.single('file'), (req, res) => {
    res.json({ message: "Файл успішно завантажено", file: req.file });
});

app.post('/api/upload-multiple', upload.array('files', 5), (req, res) => {
    res.json({ message: "Файли успішно завантажено", files: req.files });
});
```

### Завдання 6. Моніторинг стану сервера
Створено маршрут для отримання метрик використання пам'яті та часу роботи (uptime).

```javascript
// app.js
app.get('/api/status', (req, res) => {
    res.json({
        uptime: `${Math.floor(process.uptime())}s`,
        memory: process.memoryUsage(),
        cache: cache.getStats()
    });
});
```

## 4. Аналіз отриманих результатів
Впровадження Winston дозволило забезпечити надійне збереження історії подій, що є критичним для налагодження (debugging) у продакшн-середовищі. Використання Multer спростило роботу з бінарними даними, а налаштована валідація гарантує, що сервер не буде перевантажений завеликими файлами або небезпечними типами контенту. Моніторинг у реальному часі за допомогою PM2 та вбудованих засобів Node.js дозволяє вчасно виявляти "вузькі місця" продуктивності.

## 6. Контрольні питання

1. **Що таке middleware у Node.js?** Це проміжне програмне забезпечення (функція), яке має доступ до об'єктів запиту (req), відповіді (res) та наступної функції (next) у циклі. Воно може виконувати код, змінювати об'єкти запиту або завершувати цикл запит-відповідь.

2. **Яка різниця між Morgan і Winston?** Morgan використовується виключно для логування HTTP-запитів і зазвичай виводить їх у консоль. Winston — це гнучка бібліотека для логування будь-яких подій програми, яка вміє зберігати логи у файли, сортувати їх за рівнями важливості (info, warn, error) та навіть відправляти у віддалені бази даних.

3. **Як працює Multer?** Це middleware спеціально для обробки `multipart/form-data` (використовується при завантаженні файлів). Він розбирає потік вхідних даних, витягує з нього файли, зберігає їх на диск сервера і додає інформацію про них в об'єкт `req.file` (або `req.files`), щоб розробник міг далі з ними працювати.

4. **Які метрики продуктивності важливі для серверів?** Основні метрики: час відповіді (Response Time), використання оперативної пам'яті (Memory Usage), навантаження на процесор (CPU Load), кількість помилок та Uptime (час безперервної роботи).

5. **Що таке uptime?** Це показник часу (зазвичай у секундах або годинах), протягом якого сервер або окремий процес працює безперервно з моменту свого останнього запуску чи перезавантаження.
