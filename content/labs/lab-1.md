## Опис предметного середовища

### Актуальність теми
Розвиток електронної комерції робить онлайн-покупки основним способом придбання товарів, зокрема побутової техніки. Необхідність створення зручного та швидкого інтернет-магазину, який коректно відображатиметься на будь-яких пристроях (смартфонах, планшетах, десктопах), є критично важливою для успішного ведення бізнесу.

### Об'єкт та предмет роботи
- **Об'єкт:** Веб-застосунок інтернет-магазину побутової техніки.
- **Предмет:** Адаптивний користувацький інтерфейс, структура даних та алгоритми обробки замовлень.

### Бізнес-логіка роботи
- Якщо сума замовлення перевищує 15 000 грн, клієнту автоматично надається безкоштовна доставка.
- Товар не може бути доданий до кошика, якщо його залишок на складі дорівнює нулю.
- До кожної одиниці великої побутової техніки автоматично формується електронний гарантійний талон.

### Вимоги
- **Функціональні:** фільтрація техніки, кошик, оформлення замовлення, розрахунок загальної вартості, email-підтвердження.
- **Нефункціональні:** адаптивність (перебудова для екранів менше 1200px та менше 768px), час завантаження до 2 секунд, шифрування даних (HTTPS).git add .

## Хід виконання

### Крок 1: Створення файлової структури
На початковому етапі було створено кореневу папку веб-застосунку з логічно структурованими підкаталогами: `/css` (для таблиць стилів), `/js` (для скриптів) та основний файл `index.html`.

### Крок 2: Створення основного HTML-документа
У документі `index.html` було визначено основні структурні компоненти інтерфейсу: `header` (шапка з логотипом та меню), `main` (центральна частина для контенту) та `footer` (підвал з інформацією).

### Крок 3: Налаштування стилів та адаптивної сітки
У файлі `style.css` реалізовано адаптивну сітку карток товарів за допомогою технології CSS Grid. Використано медіа-запити (`@media`) для динамічної перебудови контенту залежно від ширини екрана (десктоп — 3 колонки, планшет — 2, смартфон — 1).

### Крок 4: Навігаційне меню та інтерактивність
Додано навігаційне меню, яке на мобільних пристроях трансформується у компактне «бургер-меню». Реалізовано плавне відкриття меню за допомогою JavaScript та додано hover-ефекти для кнопок і карток товарів.

### Крок 5: Робота з Git та GitHub
Було ініціалізовано локальний репозиторій Git, створено гілки (`main`, `develop` та тематичні), виконано коміти з обґрунтованими повідомленнями та оформлено файл `README.md`. Фінальний код завантажено на GitHub.
